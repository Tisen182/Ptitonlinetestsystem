Building a user-friendly and easy-to-use interface for an online multiple-choice test system:
General Requirements:

• Use HTML, CSS, and JavaScript to design and develop the interface.

• The interface should be beautiful, user-friendly, and use the PTIT interface and colors.

• Support for a responsive interface across different devices is encouraged.

System Pages

5.1. Regular Users

a. Login and Registration Page:

• Requirements:

o Login form with fields: Username and Password.

o New registration form with fields: Username, Email, Password, and Confirm Password.

o Validate login and registration information using JavaScript.

o Allow users to log in and register new accounts (demo, no need to call the backend yet).

b. Main Page:

• Requirements:

o Display a list of exams: Practice, Midterm, Final, etc.

• Exams are categorized by status: freely accessible or requiring a specific time.

• Provide a search and filter feature for exams by name or status.

• Users can select an exam to start.
c. Exam Page:
• Requirements:

• Display questions, with multiple-choice options for each question.

• Have a timer for each exam.

• A "Submit" button to send the exam.

• Display a notification when time runs out or after submission.

d. Results Page:

• Requirements:

• Display the number of correct answers and the total number of questions.

• Calculate and display the score.

• Allow users to review their answers and correct answers.

• Users can view the exam results and detailed answers.

• Fix data.

• Statistics are required.
